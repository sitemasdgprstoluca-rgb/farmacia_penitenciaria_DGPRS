🔴 CRÍTICO - Probabilidad ALTA de ocurrir:

  • Servidor dormido cuando un centro intente acceder → Espera de 60s
  • Sistema se cae con 5-10 usuarios simultáneos → Servicio no disponible
  • Sin redundancia: 1 error = todo el sistema cae
  • Sesiones se pierden con cada redeploy
  • Queries repetitivas sin caché → Base de datos saturada

🟡 ALTO - Impacto en experiencia de usuario:

  • Respuestas lentas (>2 segundos) en dashboard
  • Timeouts en reportes con muchos datos
  • Imposibilidad de escalar en picos de uso

❓ ¿Puedo volver a Free si no funciona?
   ✅ SÍ - Puedes downgrade en cualquier momento (cobro prorrateado)

   ⚠️  Altamente recomendado para múltiples instancias
      Sin Redis, las sesiones no se comparten entre instancias

❓ ¿Cuánto tarda en aplicarse el autoscaling?
   ✅ 30-60 segundos después de detectar carga alta

   ⚠️  El autoscaling escala hasta maxInstances (5), luego puede haber lentitud
      Recomendado upgrade a Pro para 100+ usuarios

   ✅ NO - El frontend es estático, plan Free es suficiente
      Solo el backend necesita escalabilidad
