# 🚀 Render Deployment Instructions

## Problem Solved
The build was failing because Render detected the root `requirements.txt` file and tried to run Django commands from the root directory instead of the `backend/` directory.

## Solution Implemented
Created root-level build and start scripts that automatically redirect to the `backend/` directory.

---

## 📋 Deployment Options

### Option 1: Manual Web Service (Quick Fix)
If you already created the service manually in the Render Dashboard:

1. **Update Service Settings:**
   - Go to your service in the Render Dashboard
   - Navigate to **Settings**
   - Update the following:

   **Build Command:**
   ```bash
   chmod +x build.sh && ./build.sh
   ```

   **Start Command:**
   ```bash
   chmod +x start.sh && ./start.sh
   ```

2. **Trigger Manual Deploy:**
   - Go to "Manual Deploy" tab
   - Click "Deploy latest commit"

### Option 2: Blueprint Deployment (Recommended)
Use the infrastructure-as-code approach with render.yaml:

1. **Delete Existing Service** (if created manually)
   - Go to Render Dashboard
   - Delete the existing `farmacia-api` service

2. **Create New Blueprint:**
   - Click **"New" → "Blueprint"**
   - Connect your GitHub repository
   - Render will automatically detect `render.yaml`
   - Review the configuration
   - Click **"Apply"**

3. **Configure Environment Variables:**
   Set these required variables in the Dashboard:
   ```
   SECRET_KEY=<auto-generated>
   DATABASE_URL=postgresql://user:pass@host:port/db
   ALLOWED_HOSTS=farmacia-api.onrender.com
   CORS_ALLOWED_ORIGINS=https://farmacia-front.onrender.com
   CSRF_TRUSTED_ORIGINS=https://farmacia-front.onrender.com,https://farmacia-api.onrender.com
   ```

## 🔧 Configuration Files

### Root Level (for auto-detection)
- ✅ `build.sh` - Redirects to backend build
- ✅ `start.sh` - Redirects to backend start
- ✅ `requirements.txt` - Mirror of backend dependencies + gunicorn

### Backend Directory (for Blueprint)
- ✅ `backend/build.sh` - Actual build script
- ✅ `backend/requirements.txt` - Django dependencies
- ✅ `render.yaml` - Infrastructure configuration

## 📝 Environment Variables Checklist

### Required (Must Configure)
- [ ] `SECRET_KEY` - Django secret (auto-generate in Render)
- [ ] `DATABASE_URL` - PostgreSQL connection string
- [ ] `ALLOWED_HOSTS` - Your Render domain
- [ ] `CORS_ALLOWED_ORIGINS` - Frontend URL
- [ ] `CSRF_TRUSTED_ORIGINS` - Frontend + Backend URLs

### Optional (Pre-configured)
- [x] `DEBUG=False`
- [x] `ENFORCE_HTTPS=True`
- [x] `PYTHON_VERSION=3.12.0`
- [x] `LOG_LEVEL=INFO`

## 🗄️ Database Setup (Supabase PostgreSQL)

1. **Create Supabase Project:**
   - Go to https://supabase.com
   - Create a new project
   - Wait for database to provision

2. **Get Connection String:**
   - Go to Project Settings → Database
   - Copy the connection pooler URL (port 6543)
   - Format: `postgresql://postgres.[PROJECT-REF]:[PASSWORD]@aws-0-us-west-1.pooler.supabase.com:6543/postgres`

3. **Add to Render:**
   - Paste as `DATABASE_URL` environment variable

## 🐛 Troubleshooting

### Build Fails: "No such file or directory"
- **Cause:** Scripts don't have execute permissions
- **Fix:** Render build command should include `chmod +x build.sh`

### Build Fails: "manage.py not found"
- **Cause:** Running from wrong directory
- **Fix:** Ensure build.sh includes `cd backend`

### Start Fails: "gunicorn: command not found"
- **Cause:** gunicorn not in requirements.txt
- **Fix:** Already fixed - gunicorn==22.0.0 added to root requirements.txt

### Database Connection Error
- **Cause:** DATABASE_URL not set or incorrect
- **Fix:** Double-check Supabase connection string

### 500 Error on API Calls
- **Causes:** 
  - ALLOWED_HOSTS doesn't include your domain
  - CORS_ALLOWED_ORIGINS doesn't include frontend
  - CSRF_TRUSTED_ORIGINS missing
- **Fix:** Set all three environment variables correctly
